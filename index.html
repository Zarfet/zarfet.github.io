Where technical quality meets user experience

My curiosity about how things worked behind the screens led me to Quality Assurance. That transition became my edge.

Most QA analysts test functionality. I test functionality AND experience. I read code, understand system architecture, and use modern testing tools, but my differentiator is catching issues where technical execution meets user flow.

I've spent 10+ years leading QA strategies for complex digital products: e-commerce platforms handling millions of transactions, fintech apps, media applications. I work with Selenium, Cypress, and agile frameworks, but the real value is my hybrid perspective.

A feature can pass every functional test and still create a terrible user experience. I'm the person who catches that disconnect.

If you need QA that goes beyond "does it work?" to "does it make sense?", let's talk.
